Welcome to Last Modified Timestamp

Track when web pages were last updated with this simple, powerful extension

Why We Built This Extension

Don't you hate it when you're on a webpage and you're unsure if the information is new or old?

Sadly, few websites clearly display when their content was last updated or modified. This can be frustrating when you need to know if you're looking at current information.

Last Modified Timestamp solves this problem by allowing you to easily check when a webpage was last updated. Simply click the extension icon to find out, giving you confidence in the content you're reading.

For the best experience, we recommend pinning this extension to your toolbar for easy access.

How the Extension Works

We check every possible location where a timestamp might be stored, following a logical sequence from the most accurate sources to less reliable ones:

1

Meta last-modified Tag

First, we look for the official HTML meta tag specifically designed for last-modified timestamps.

2

Other Meta Timestamps

We check for any meta tags with relevant "updated" or "modified" timestamps, including Open Graph (OG) tags/properties. If no match is found, we search again for a "published" timestamp.

3

Schema.org Structured Data

For pages with structured data (schema), if the type is Article, BlogPosting, or similar, we look for a lastModified, dateUpdated, or datePublished property/value.

4

HTML Content Patterns

We scan the page HTML for modified, updated, or published patterns in the content that contain a timestamp.

5

HTTP Last-Modified Header

Finally, we check for this response header. If the value is the same as the current time or if the header doesn't exist, we treat it as "not found".

Note: Some webpages simply don't provide timestamp information in any form. In these cases, even our thorough checking can't find what doesn't exist.
